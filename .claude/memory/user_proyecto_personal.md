Pedro Zambudio (`pjzambudio@gmail.com`) está desarrollando BuenChollo como
proyecto personal a largo plazo. El TFM es la presentación inmediata, pero
el objetivo real es lanzarlo en producción y hacerlo crecer (tiene "muchas
ideas" para nuevas features).

Implicación práctica para cualquier decisión técnica:

- Priorizar siempre **calidad de producción y escalabilidad real**, no sólo
  aprobar el módulo del máster.
- Si detectas algo que en producción dará problemas, **arreglarlo ya** aunque
  el TFM no lo exija. "Quiero todo perfecto".
- Tono y nivel de calidad esperado: **profesional senior**, no estudiante.
- Cuando haya trade-off entre simplicidad-MVP y solidez-largo-plazo,
  inclinarse al largo plazo (siempre que no sea sobreingeniería absurda).
